## Начало работы

### Быстрый старт с PostgreSQL (дамп из репозитория)

Чтобы сразу работать в интерфейсе без ручного наката всех миграций на пустую БД:

1. Создайте базу PostgreSQL (например, `nikacrm`).
2. Импортируйте файл **`database/bootstrap/nikacrm_public_sanitized.sql`** (команды и пояснения — в [database/bootstrap/README.md](../database/bootstrap/README.md)).
3. В `.env` укажите `DB_DRIVER=postgres` и корректный `DATABASE_URL`.
4. Установите зависимости и запустите приложение (`python run.py` или Docker). В дампе уже отмечены применённые версии миграций Postgres **001–008**; демо-логины и пароли см. в README bootstrap.

### Вход в систему

1. Откройте браузер и перейдите по адресу системы (например, `http://127.0.0.1:5000`)
2. На странице входа введите:
   - **Имя пользователя** - ваш логин
   - **Пароль** - ваш пароль
3. Нажмите кнопку "Войти"

**Важно:** После 5 неудачных попыток входа доступ будет временно заблокирован на 1 минуту (защита от brute-force атак).

### Главная страница (Дашборд)

После входа вы увидите главную страницу с:
- **Статистикой по заявкам** - количество заявок по статусам
- **Графиком заявок по дням** - последние 7 дней
- **Топ-5 типов устройств** - наиболее частые типы устройств
- **Топ-5 менеджеров** - менеджеры с наибольшим количеством заявок
- **Последние 10 заявок** - недавно созданные заявки

## Работа с заявками

### Создание новой заявки

1. Перейдите в раздел **"Заявки"** → **"Новая заявка"**
2. Заполните форму:
   - **Клиент:**
     - Имя клиента (обязательно)
     - Телефон (обязательно, автоматически нормализуется)
     - Email (опционально)
   - **Устройство:**
     - Тип устройства (обязательно) - выберите из списка
     - Бренд (обязательно) - выберите из списка
     - Модель (опционально) - введите или выберите из автопоиска
     - Серийный номер (опционально)
     - Пароль от устройства (опционально)
   - **Неисправность:**
     - Выберите симптомы из списка или введите свои (теги через запятую, точку с запятой или перенос строки)
     - Система автоматически создаст новые симптомы в справочнике, если их нет
   - **Внешний вид:**
     - Выберите теги внешнего вида или введите свои (теги через запятую, точку с запятой или перенос строки)
     - Система автоматически создаст новые теги в справочнике, если их нет
   - **Дополнительно:**
     - Менеджер (обязательно)
     - Мастер (опционально)
     - Предоплата (опционально)
     - Комментарий (опционально)
3. Нажмите кнопку **"Создать заявку"**

**Особенности:**
- Если клиент с таким телефоном уже существует, он будет автоматически найден
- Модель устройства сохраняется в справочнике для быстрого выбора в будущем
- Можно заполнить форму данными из последней заявки клиента (кнопка "Заполнить из последней заявки")

### Просмотр списка заявок

Перейдите в раздел **"Заявки"** → **"Все заявки"**

#### Представления

Система поддерживает три представления:

1. **Реестр** - таблица со всеми заявками
2. **Канбан** - карточки заявок по статусам
3. **Журнал** - упрощенный список заявок

Переключение между представлениями через кнопки в верхней части страницы.

#### Фильтрация и поиск

- **Поиск** - введите текст в поле поиска (поиск по клиенту, телефону, номеру заявки, модели)
- **Фильтр по статусу** - выберите статус из выпадающего списка
- **Фильтр по менеджеру** - выберите менеджера
- **Фильтр по мастеру** - выберите мастера
- **Фильтр по датам** - выберите период (дата начала и дата окончания)

**Особенности:**
- Отображаются **все заявки**, включая заявки без устройств
- Поля "Устройство" и "Бренд" являются ссылками на историю устройства (если устройство указано)
- Неисправности и внешний вид отображаются в виде цветных тегов
- Данные загружаются из нормализованных таблиц для быстрого поиска и аналитики

**Нормализация данных:**
Система использует нормализованную структуру базы данных:
- Модели устройств связаны через внешние ключи (FK)
- Симптомы и теги внешнего вида связаны через связующие таблицы (many-to-many)
- Это обеспечивает целостность данных, быстрый поиск и возможность аналитики

#### Сортировка

Нажмите на заголовок колонки для сортировки:
- Первый клик - сортировка по возрастанию (ASC)
- Второй клик - сортировка по убыванию (DESC)
- Третий клик - сброс сортировки

#### Настройка отображения колонок

Нажмите кнопку **«Колонки»** (рядом с выбором «Показать записей») для включения или выключения колонок. Выбранный набор сохраняется в браузере. Порядок колонок можно менять перетаскиванием заголовков — он тоже сохраняется.

### Просмотр деталей заявки

1. Нажмите на заявку в списке или перейдите по ссылке
2. На странице заявки вы увидите:
   - **Информацию о клиенте** - имя, телефон, email
   - **Информацию об устройстве** - тип, бренд, модель, серийный номер
   - **Статус заявки** - текущий статус с цветовой индикацией
   - **Услуги** - список услуг с ценами
   - **Запчасти** - список запчастей с ценами
   - **Оплаты** - история оплат
   - **Комментарии** - комментарии к заявке
   - **История изменений** - лог всех изменений

### Редактирование заявки

1. Откройте страницу заявки
2. Нажмите кнопку **"Редактировать"**
3. В модальном окне измените необходимые поля:
   - **Тип устройства** - выберите из списка
   - **Бренд** - выберите из списка
   - **Модель** - введите или выберите из автопоиска (новые модели автоматически сохраняются)
   - **Серийный номер** - обновите при необходимости
   - **Пароль от устройства** - обновите при необходимости
   - **Внешний вид** - выберите теги или введите свои
   - **Неисправность** - выберите теги или введите свои
   - **Менеджер** - выберите из списка
   - **Мастер** - выберите из списка
   - **Предоплата** - обновите сумму
   - **Комментарий** - обновите комментарий
4. Нажмите **"Сохранить"**

**Особенности:**
- Поле "Модель" поддерживает автопоиск и автоматическое создание новых моделей
- Все поля опциональны - обновляются только измененные поля
- При обновлении устройства также обновляется связанное устройство (если указано)

### Изменение статуса заявки

Статус заявки можно изменить двумя способами:

#### 1. На странице заявки

1. На странице заявки найдите кнопку со статусом (цветная кнопка в верхней части)
2. Нажмите на кнопку и выберите новый статус из выпадающего списка
3. Статус изменится автоматически
4. Появится **цветное уведомление** в центре верхней части страницы с информацией об изменении статуса

#### 2. В списке заявок

1. В таблице заявок найдите кнопку со статусом в колонке "Статус"
2. Нажмите на кнопку и выберите новый статус
3. Статус изменится автоматически
4. Появится **цветное уведомление** в центре верхней части страницы с информацией об изменении статуса

**Доступные статусы:**
- **Новая** - только что созданная заявка (желтый цвет)
- **В работе** - заявка в процессе выполнения (синий цвет)
- **Готова** - заявка выполнена, ожидает выдачи (зеленый цвет)
- **Закрыта** - заявка закрыта (серый цвет)

**Начисление зарплаты при смене статуса:**
- Если у статуса включён флаг «Начисляет зарплату» (например, «Закрыто»), зарплата по заявке начисляется **один раз** — при первом переходе в этот статус. Повторная смена статуса (например, «Закрыто → В работе → Закрыто») не создаёт повторных начислений и не переносит дату начислений. Это защищает от дублирования зарплаты при случайном переключении статуса.

**Особенности:**
- Уведомление автоматически исчезает через 3 секунды
- Цвет уведомления соответствует цвету выбранного статуса
- Изменение статуса логируется в истории действий

### Добавление услуг к заявке

1. На странице заявки найдите раздел **"Услуги"**
2. Нажмите кнопку **"Добавить услугу"**
3. Выберите услугу из списка
4. Укажите количество и цену
5. Нажмите **"Добавить"**

### Добавление запчастей к заявке

1. На странице заявки найдите раздел **"Запчасти"**
2. Нажмите кнопку **"Добавить запчасть"**
3. Введите название или артикул для поиска
4. Выберите запчасть из результатов поиска
5. Укажите количество и цену
6. Нажмите **"Добавить"**

**Примечание:** При добавлении запчасти автоматически списывается количество со склада.

### Добавление оплаты

1. На странице заявки найдите раздел **"Оплаты"**
2. Нажмите кнопку **"Добавить оплату"**
3. Укажите:
   - Сумму оплаты
   - Тип оплаты (наличные, карта, перевод)
   - Комментарий (опционально)
4. Нажмите **"Добавить"**

**Важно:**
- При добавлении обычной оплаты автоматически создается кассовая операция
- Оплата отображается в разделе **"Финансы"** → **"Касса"**
- Оплата учитывается во всех финансовых отчетах
- Прогресс оплаты ограничен 100% (переплата не учитывается в прогрессе)

### Возврат оплаты

Для возврата оплаты по заявке (требуется роль менеджера):

1. На странице заявки в разделе **"Оплаты"** найдите нужную оплату
2. Нажмите кнопку **"Возврат"** рядом с оплатой
3. Укажите:
   - Сумму возврата (не больше оставшейся суммы оплаты)
   - Причину возврата (обязательно)
   - Создать чек возврата (опционально)
4. Нажмите **"Вернуть"**

**Особенности:**
- Создаётся возвратный платеж (`kind='refund'`)
- Создаётся расходная кассовая операция
- Сумма возврата ограничена оставшейся суммой оригинальной оплаты
- Опционально создаётся фискальный чек возврата

### Депозит в заявках

Депозитные операции в интерфейсе заявки отключены:
- оплата с депозита,
- возврат в депозит,
- перевод переплаты в депозит.

### Отмена оплаты

Для отмены оплаты (требуется роль менеджера):

1. На странице заявки в разделе **"Оплаты"** найдите нужную оплату
2. Нажмите кнопку **"Отменить оплату"**
3. Укажите причину отмены (обязательно)
4. Нажмите **"Отменить"**

**Особенности:**
- Оплата помечается как отменённая (не удаляется)
- Создаётся сторно-операция в кассе
- История платежа сохраняется для аудита
- Отменённые платежи не учитываются в расчётах

### Добавление комментария

1. На странице заявки найдите раздел **"Комментарии"**
2. Введите текст комментария
3. Нажмите **"Добавить комментарий"**

### Скрытие заявки

Если заявка больше не актуальна, но не должна быть удалена:

1. На странице заявки нажмите кнопку **"Скрыть заявку"**
2. Укажите причину скрытия
3. Заявка будет скрыта из основных списков, но останется в базе данных

## Работа с клиентами

### Просмотр списка клиентов

Перейдите в раздел **"Клиенты"** → **"Все клиенты"**

**Особенности:**
- По умолчанию таблица отсортирована по ID в порядке убывания (новые клиенты сверху)
- Один поисковый блок (дубликат удален)

#### Поиск клиентов

- Введите текст в поле поиска (поиск по имени, телефону, email)
- Нажмите Enter или кнопку поиска

#### Сортировка

Нажмите на заголовок колонки для сортировки по:
- ID (по умолчанию - убывание, новые клиенты сверху)
- Имени
- Телефону
- Email
- Дате создания

### Просмотр деталей клиента

1. Нажмите на клиента в списке
2. На странице клиента вы увидите:
   - **Контактную информацию** - имя, телефон, email
   - **Депозит (кошелёк)** - текущий баланс депозита клиента (если есть)
   - **Устройства клиента** - список всех устройств
   - **Историю заявок** - все заявки по клиенту
   - **Все продажи** - объединенный список продаж из заявок и магазина:
     - Продажи из заявок отображаются с пометкой "Заявка #X"
     - Продажи из магазина отображаются с пометкой "Магазин #X"
     - Ссылки на детали продажи (заявку или чек магазина)
     - Дата, сумма, услуги и запчасти

### Депозит (кошелёк) клиента

Депозит клиента - это виртуальный кошелёк, на котором хранятся средства клиента для будущих оплат.

**Как пополняется депозит:**
- Ручное пополнение через API (для администраторов)

**Как используется депозит:**
- Используется для операций кошелька клиента вне интерфейса заявки

**Просмотр истории операций:**
- История операций с депозитом доступна через API
- Все операции логируются в таблице `customer_wallet_transactions`
- Каждая операция содержит: тип (пополнение/списание), сумму, источник, комментарий, дату

### Редактирование клиента

1. Откройте страницу клиента (`/clients/<id>`)
2. Нажмите кнопку **"Редактировать"**
3. В модальном окне редактирования вы можете изменить:
   - **Имя клиента** (обязательно)
   - **Телефон** (обязательно)
   - **Email** (опционально)
   - **Пароль для портала** (опционально):
     - Введите новый пароль для доступа клиента к личному кабинету
     - Минимум 4 символа (буквы и цифры)
     - Оставьте пустым, чтобы не менять существующий пароль
     - Нажмите **"Удалить пароль"** для отключения доступа к порталу
     - Индикатор показывает статус доступа к порталу

**Особенности управления паролем портала:**
- При установке нового пароля администратором клиент должен будет сменить его при следующем входе
- Если пароль удален, клиент не сможет войти в портал до установки нового пароля
- Новые клиенты автоматически получают сгенерированный пароль при создании

### Создание нового клиента

1. Перейдите в раздел **"Клиенты"** → **"Все клиенты"**
2. Нажмите кнопку **"Добавить клиента"**
3. Заполните форму:
   - **Имя** (обязательно)
   - **Телефон** (обязательно)
   - **Email** (опционально)
4. Нажмите **"Сохранить"**

**Важно:** При создании нового клиента система **автоматически генерирует пароль** для доступа к порталу. Пароль состоит из 8 символов (буквы и цифры) и будет доступен администратору для передачи клиенту.

### Редактирование клиента

1. Откройте страницу клиента (`/clients/<id>`)
2. Нажмите кнопку **"Редактировать"**
3. В модальном окне редактирования вы можете изменить:
   - **Имя клиента** (обязательно)
   - **Телефон** (обязательно)
   - **Email** (опционально)
   - **Пароль для портала** (опционально):
     - Введите новый пароль для доступа клиента к личному кабинету
     - Минимум 4 символа (буквы и цифры)
     - Оставьте пустым, чтобы не менять существующий пароль
     - Нажмите **"Удалить пароль"** для отключения доступа к порталу
     - Индикатор показывает статус доступа к порталу
4. Нажмите **"Сохранить изменения"**

**Особенности управления паролем портала:**
- При установке нового пароля администратором клиент должен будет сменить его при следующем входе
- Если пароль удален, клиент не сможет войти в портал до установки нового пароля

## Финансы

Финансовый модуль обеспечивает учёт денежных потоков и аналитику.

### Касса

Перейдите в раздел **"Финансы"** → **"Касса"**

На странице кассы вы увидите:
- **Итоговый баланс** за период
- **Остаток на начало**
- **Приход** и **Расход** за период
- Таблицу всех кассовых операций
- Блок с остатками по способам оплаты (Наличные, Перевод и др.) на конец периода

**Особенности баланса:**
- Баланс может быть отрицательным
- При отрицательном балансе карточка обводится красной рамкой
- Появляется предупреждающий бейдж "⚠️ Отрицательный"
- Сумма отображается красным цветом

**Автоматическая синхронизация:**
- Все оплаты из заявок автоматически создают кассовые операции
- Предоплаты при создании заявки автоматически записываются в кассу
- При удалении оплаты автоматически удаляется соответствующая кассовая операция
- Оплаты отображаются в кассе с описанием (номер заявки, клиент)

**Ссылки в списке операций:**
- **«Заявка #N»** — ведёт на страницу заявки (открывается по UUID или по внутреннему номеру заявки).
- **«Чек #N»** — для операций по оплате ведёт на страницу чека (детали оплаты); для ручных операций отображается «Операция #N» без ссылки.

#### Перевод между кассами

Для логического перемещения денег между способами оплаты (например, из Переводов в Наличные) используйте кнопку **«Перевод между кассами»**:

1. Нажмите кнопку **«Перевод между кассами»**.
2. В форме выберите:
   - **Списать с** — текущий источник (например, Перевод).
   - **Зачислить на** — целевой способ (чаще всего Наличные).
   - **Сумма** — размер перевода.
   - **Дата** и **Примечание** (по желанию).
3. Нажмите **«Выполнить перевод»**.

Система создаёт две кассовые операции:
- **Расход** по способу «Списать с».
- **Приход** по способу «Зачислить на».

При этом:
- Итоговый баланс кассы не меняется.
- «Перевод между кассами» **не попадает** в показатели «Приход за период» и «Расход за период», но корректирует остатки по отдельным способам оплаты.

#### Дисциплина по Наличным и Переводам

- Для ручных операций **Расход** по способам **Наличные** и **Перевод** действует защита:
  - Система проверяет доступный остаток по выбранному способу оплаты на дату операции.
  - Если пытаетесь списать больше, чем доступно, операция не будет создана, и вы увидите сообщение об ошибке.
- В карточке **«Наличные»** отображается подсказка:
  - При отрицательном остатке система показывает блок **«Доложить в наличку: X ₽»** — это сумма, которой не хватает до нулевого остатка по наличным операциям (учитываются только реальные приходы/расходы, без внутренних переводов).

#### Фильтрация по периоду

Используйте фильтр периода:
- **Сегодня** - операции за сегодня
- **Вчера** - операции за вчера
- **С начала недели** - операции с понедельника
- **За месяц** - операции за текущий месяц
- **Прошлый месяц** - операции за прошлый месяц
- **За год** - операции за текущий год
- **Выбрать период** - произвольный диапазон дат

#### Добавление операции

1. Нажмите кнопку **"Добавить операцию"**
2. Выберите тип операции:
   - **Приход** - поступление денег
   - **Расход** - траты
3. Выберите категорию (статью)
4. Укажите сумму
5. Выберите способ оплаты
6. Добавьте описание (опционально)
7. Нажмите **"Сохранить"**

#### Удаление операции

1. В списке операций нажмите кнопку удаления
2. Подтвердите удаление

**Примечание:** Автоматические операции (от продаж через магазин) также можно удалить.

### Отчет по зарплате

Перейдите в раздел **"Отчеты"** → **"Зарплата"**

Отчет показывает:
- **Карточки «Итоги»** — Выручка (сумма поступлений по заявкам и магазину), Итого руководителю (выручка − себестоимость − зарплата), Всего начислений, Сумма начислений, Прибыль, Сотрудников. Для каждой карточки в интерфейсе отображается краткий комментарий с формулой расчёта. Период задаётся фильтром (по умолчанию заканчивается сегодня).
- **Таблицу начислений** по мастерам и менеджерам с колонками: Дата, Заявка (#ID), Сотрудник, Роль, Выручка, Прибыль, Начислено, Руководителю, Правило. Выручка и прибыль считаются **на уровне заявки** — по одной заявке у менеджера и мастера отображаются одинаковые значения (фактическая выручка и прибыль заявки).

Для правил типа **`fixed` (фиксированная сумма)**, применяемых к услугам и запчастям, начисление зарплаты учитывает **количество в позиции** как в заявках, так и в магазине: если услуга/товар указан с `Кол-во = 2`, фиксированная часть зарплаты умножается на 2.

## Чат сотрудников

Внутренний чат для сотрудников доступен **на всех основных страницах** CRM (плавающая кнопка с иконкой сообщений). Его видят пользователи с соответствующими правами (заявки, склад, финансы, магазин, зарплата и т.д.).

### Подпись на этом ПК

1. Откройте панель чата.
2. В поле **«Кто вы (на этом ПК)»** введите понятную подпись (например, «Менеджер Ольга») и нажмите **«Сохранить»**.  
   Подпись сохраняется в браузере и отображается в сообщениях; так коллеги отличают одинаковые учётные записи с разных компьютеров.

### Сообщения и вложения

- Текст в поле ввода, отправка кнопкой или сочетанием клавиш (как в интерфейсе).
- Вложения: кнопка прикрепления файла; действуют **лимиты** по числу файлов, размеру и длине текста (подсказка под полем ввода).
- У своих сообщений доступны **«Изм.»** и **«Удал.»**; у всех — **реакции** (эмодзи).

### «Прочитано» (курсор чтения)

- Когда панель чата **открыта**, система периодически сохраняет, до какого сообщения вы «дочитали».
- Под сообщениями других сотрудников может отображаться строка **«Прочитали: …»** и кнопка **«+N»** — полный список по запросу.
- Обновление у коллег в реальном времени: после сохранения курсора рассылается событие по WebSocket, список «Прочитали» подтягивается без перезагрузки страницы.

### Звук и заголовок вкладки

- При входящем сообщении от другого пользователя может проиграть **звук** (после первого действия на странице — из‑за политики браузера).
- Если вкладка **не активна**, заголовок может показывать **напоминание о новом сообщении** (сбрасывается при возврате на вкладку).

### Уведомления 🔔 (без Web Push)

- Кнопка **🔔** в шапке панели чата включает **системные уведомления ОС** (не путать с колокольчиком уведомлений CRM в шапке сайта).
- Первый раз браузер запросит **разрешение**; без «Разрешить» баннеры не появятся.
- Уведомление обычно **не показывается**, если вы уже смотрите **открытую** панель чата на активной вкладке; при **закрытой** панели или неактивной вкладке — показывается (если включено 🔔).
- **Shift+клик по 🔔** — проверка: тестовое уведомление и подсказка в строке статуса чата. Если тест «отправлен», но баннера нет — смотрите **Центр уведомлений Windows (Win+N)** и настройки Chrome для сайта.

### Web Push 📡 (вкладка может быть закрыта)

- Кнопка **📡** видна только если на сервере настроены **VAPID-ключи** и установлен пакет **pywebpush**.
- Включение: клик по **📡**, разрешение уведомлений, регистрация в браузере. Отключение — повторный клик (отписка на сервере и в браузере).
- Работает по **HTTPS** или на **localhost / 127.0.0.1**. Отправителю своих сообщений push **не** дублируется на его же устройства.
- Администратору: переменные `STAFF_CHAT_VAPID_*` в `.env`, миграции БД для таблицы подписок; см. `docs/API.md` и `.env.example`.

### Технические замечания

- Реальное время: **Socket.IO** (`/staff-chat`); при недоступности сокета возможен ограниченный режим с периодическим опросом.
- Отдельно от чата остаётся **система уведомлений CRM** (заявки, задачи, упоминания) — см. раздел [Система уведомлений](#система-уведомлений).

## Расширенная система комментариев

### Добавление комментария с упоминанием

1. В деталях заявки найдите форму комментария
2. Введите текст комментария, используя `@username` для упоминания пользователя:

   @admin, пожалуйста, проверьте это устройство

- **Упоминания** выделены цветом и кликабельны
- **Внутренние комментарии** помечены бейджем "Internal"
- **Файлы** отображаются под комментарием с возможностью скачивания

## Система задач и дедлайнов

### Создание задачи

1. В деталях заявки нажмите кнопку **"Задачи"** (справа от кнопки "Комментарии")
2. Откроется боковая панель с задачами
3. Нажмите **"+ Создать задачу"**
4. Заполните форму:
   - **Название** (обязательно)
   - **Описание** (опционально)
   - **Исполнитель** - выберите из списка пользователей
   - **Дедлайн** - выберите дату и время
   - **Приоритет** - low, medium, high, urgent
   - **Чек-лист** - добавьте пункты для выполнения (опционально)
5. Нажмите "Сохранить"

### Управление задачами

- **Просмотр задач:**
  - Все задачи заявки отображаются в боковой панели
  - Счетчик задач показывает количество задач
- **Редактирование задачи:**
  - Нажмите на задачу для редактирования
  - Измените статус, приоритет, дедлайн
- **Отметка чек-листа:**
  - Отмечайте пункты чек-листа по мере выполнения
- **Удаление задачи:**
  - Нажмите кнопку удаления (только создатель или администратор)

### Статусы задач

- **Todo** - новая задача
- **In Progress** - задача в работе
- **Done** - задача выполнена
- **Cancelled** - задача отменена

### Просроченные задачи

Просроченные задачи (дедлайн прошел) выделяются красным цветом и отображаются в отдельном списке.

3. **Требует право:** `view_reports`

### KPI метрики

На странице аналитики отображаются 4 основные метрики:

1. **Среднее время ремонта** - среднее количество дней от создания до закрытия заявки
2. **Повторные обращения** - процент клиентов, которые обращались повторно
3. **Средний чек** - средняя сумма заявки
4. **Конверсия заявок** - процент заявок, которые были оплачены

### Графики

1. **Динамика продаж** (линейный график):
   - Показывает выручку по дням/неделям/месяцам
   - Можно выбрать период: 7, 30, 90, 365 дней

2. **Продажи по категориям** (doughnut chart):
   - Распределение продаж по категориям услуг/товаров

3. **Воронка заявок** (bar chart):
   - Количество заявок по статусам
   - Показывает конверсию на каждом этапе

4. **Рентабельность** (bar chart):
   - Маржинальность по категориям

### Фильтрация по периодам

Выберите период в правом верхнем углу:
- 7 дней
- 30 дней (по умолчанию)
- 90 дней

## Личный кабинет клиента

### Автоматическая генерация пароля

При создании нового клиента (через создание заявки или через `/clients`) система **автоматически генерирует пароль** для доступа к порталу:
- Пароль состоит из 8 символов (буквы и цифры)
- Пароль сохраняется в системе и доступен администратору
- При первом входе клиент **обязан сменить пароль** на свой

### Вход в портал

1. Перейдите по адресу `/portal/login`
2. Введите:
   - **Телефон клиента** - номер телефона, указанный при регистрации
   - **Пароль портала** - пароль, установленный администратором или сгенерированный автоматически

**Важно:** 
- Пароль портала **отдельный** от пароля CRM системы
- При первом входе система **обязательно потребует сменить пароль** на новый
- После смены пароля вход будет проходить с новым паролем

### Смена пароля при первом входе

При первом входе в портал клиент увидит форму смены пароля:

1. Введите **текущий пароль** (выданный администратором)
2. Введите **новый пароль** (минимум 4 символа, буквы и цифры)
3. Подтвердите новый пароль
4. Нажмите **"Сменить пароль и войти"**

После успешной смены пароля клиент автоматически войдет в портал.

### Настройка доступа для клиента (для администраторов)

**В CRM системе:**

1. Откройте детали клиента (`/clients/<id>`)
2. Нажмите кнопку **"Редактировать"**
3. В форме редактирования найдите раздел **"Пароль для портала"**:
   - **Установить новый пароль:** Введите новый пароль в поле (минимум 4 символа)
   - **Удалить пароль:** Нажмите кнопку "Удалить пароль" для отключения доступа к порталу
   - **Статус доступа:** Индикатор показывает, включен ли доступ к порталу

**Примечания:**
- При установке пароля администратором клиент снова должен будет сменить его при следующем входе
- Если пароль удален, клиент не сможет войти в портал до установки нового пароля

### Страницы портала

1. **Дашборд** (`/portal/dashboard`):
   - Краткая статистика по заявкам
   - Баланс кошелька
   - Последние заявки

2. **Мои заявки** (`/portal/orders`):
   - Полный список всех заявок клиента
   - Статусы заявок
   - Суммы и даты

3. **Платежи** (`/portal/payments`):
   - История всех платежей
   - Способы оплаты
   - Даты и суммы

4. **Кошелёк** (`/portal/wallet`):
   - Текущий баланс депозита
   - История операций (пополнения, списания)
   - Детализация транзакций

### Безопасность портала

- **Отдельная аутентификация** - не связана с CRM системой
- **Ограниченный доступ** - клиент видит только свои данные
- **Rate limiting** - защита от злоупотреблений (5 попыток входа в минуту)
- **Обязательная смена пароля** - при первом входе клиент должен установить свой пароль
- **CSRF защита** - все формы защищены от CSRF атак
- **Автоматическая генерация паролей** - для новых клиентов пароли генерируются автоматически

## Безопасность и права доступа

### Как работает доступ

- Доступ определяется **правами (permissions)**, а не только ролью.
- Меню и разделы интерфейса **скрываются по правам**: если у пользователя нет права на модуль, он не увидит пункт меню.
- Даже если вручную открыть URL, сервер проверит права и заблокирует доступ.

### Важное ограничение для сотрудников

Для сотрудников (мастер/менеджер) используются кастомные роли вида `master_{id}` / `manager_{id}`.

**Запрещено** выдавать сотрудникам права:
- `manage_users`
- `manage_settings`

Эти права доступны только администратору (роль `admin`) и предназначены для управления пользователями и системными настройками.

### Новые права модулей (2026-01-20)

- Финансы: `view_finance`, `manage_finance`
- Магазин: `view_shop`, `manage_shop`
- Логи действий: `view_action_logs`
- Статусы: `manage_statuses`

### Как проверить права (для администратора)

- Список прав и назначений ролям:
  - `python scripts/audit_permissions.py`
- Проверка конкретного пользователя:
  - `python scripts/check_user_permissions.py <username>`

### Роли пользователей

Система поддерживает роли и права доступа:

1. **Viewer (Просмотр)** — только просмотр данных, без возможности редактирования
2. **Master (Мастер)** — работа с заявками (в рамках назначенных прав)
3. **Manager (Менеджер)** — расширенные права (в рамках назначенных прав)
4. **Admin (Администратор)** — полный доступ ко всем функциям системы

Дополнительно возможны **индивидуальные роли** для сотрудников:
- `master_{id}`
- `manager_{id}`

### Защита от злоупотреблений

Система защищена от:
- **Brute-force атак** - ограничение попыток входа (5 в минуту)
- **Злоупотребления API** - rate limiting на API endpoints
- **CSRF атак** - защита всех форм токенами
- **SQL injection** - параметризованные запросы

### Рекомендации по безопасности

1. **Используйте сложные пароли** - минимум 8 символов, комбинация букв, цифр и символов
2. **Не передавайте пароли** - администратор никогда не попросит ваш пароль
3. **Выходите из системы** - особенно на общих компьютерах
4. **Сообщайте о подозрительной активности** - если заметили что-то необычное
