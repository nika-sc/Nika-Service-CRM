# Security Policy

## Supported Versions

Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please report vulnerabilities privately:

- Open a private GitHub security advisory, or
- Contact maintainers via private channels.

Do not publish exploit details in public issues before a fix is available.

## Data Safety

Never include real personal data, credentials, production dumps, or private infrastructure details in issues or pull requests.
